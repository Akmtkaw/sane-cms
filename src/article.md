# CMS って結局なんやねん

Web サイトを構成する一部のデータ (画像やテキストなど) を一元に管理するシステムのことです。

英語の Content Management System から略された用語です。

まあ、ここまではほとんどの方が「そんなの当たり前じゃん」となるのですが、触れたことのなかった私にとっては「へぇ〜」で終わっていました。

こういうのは、定義と経緯を押さえてこそ一歩理解が進みます。

## 経緯

Web というものの歴史は平成元年の CERN が構築したものが始まりで[1]、国内初の Web サイトは 1992 年の KEK による HTML ページ、商用検索の Web サイトとしては 1996 年 4 月の [Yahoo! JAPAN](https://www.yahoo.co.jp/) になります。

当時は、Web サイトやページの作成・編集のために、HTML をはじめとした言語を直接編集し、サーバーにアップロードする必要がありました。そうなると、修正が技術的な知見のある人に限られてきます。そこで、誰でも修正ができるように考えられたシステムが CMS です。サーバーやページの基本構造はそのままに、特定のコンテンツのみを専用の管理画面から変更できる仕組みになっています。技術力の高いエンジニアがシステムの開発・運用保守を担当する一方、技術に詳しくないサイト運営者でも、魅力的なコンテンツを随時更新することが可能です。

私はこう理解したのでした。

# CMS を体験してみた

直近の案件で Next.js と関わりがあるので、Next.js を使って実践したいと思い、書籍 ([文献](#文献) 2 つ目) を手に取りやってみました。

## 利用技術・ツール

- TypeScript 5.7.3

- Next.js 15.1.7

- node 22.11.0

- npm 11.1.0

- [microCMS](https://microcms.io/)

  - 日本製のヘッドレス CMS

- [Vercel](https://vercel.com/)

  - フロントエンド向けのホスティングサービス

### 補足

CMS にもいくつか種類がありますが、今回はヘッドレス CMS を利用しました。これは、Next.js で構築したアプリケーションを一般的な CMS プラットフォーム上 (主に PHP で実装される) で動かすことができないからです。ヘッドレス CMS はビュー (表示部分) がないので、フロント部分を自作する必要がありますが、逆に言えばフロントエンドとバックエンドを切り離して別で考えることができます。

# 最近の CMS 事情

ここまでで、CMS について歴史や実践を交えた理解はできたかなと思います。しかし、「最近の動向としてはどうなんだい」という疑問が出てきました。

そこで、今や日常生活と切っても切り離せないスマートフォンとの関わり、という切り口から書かれた著書 ([文献](#文献) 1 つ目) がありましたので読んでみました。本章はその要約と所感です。

## スマートフォンの普及で変わってきたこと

### Web サイトの閲覧方法

2007 年に Apple から発表された iPhone を皮切りに、現在では「携帯電話」といえばスマートフォンを指すほど、私たちの日常生活に浸透しています。Web サイトを閲覧する方法も、PC だけでなくスマートフォンやタブレットへと広がり、むしろスマートフォンからの閲覧が主流になりつつあります。スマートフォンを使えば、「いつでも」「どこでも」「すばやく」情報にアクセスできます。

### 情報へのアクセス経路

「トップページから目的のページへ」から、「SNS や検索エンジンからダイレクトに」と変わってきています。

## コンテンツファーストの考え方

スマートフォンの普及によって、CMS の運用において「コンテンツ・ファースト」という考え方が重要になります。

以前は魅力的なデザインやブランディングを重視したサイト構築が主流でしたが、現在ではユーザーにいち早く必要な情報を届けることや、次回もサイトに訪問してもらうための施策がより重要になっています。具体的には次の 4 つが重要なのでは、と感じています。

1. **正しい**情報を**素早く**公開する
2. 関連するコンテンツをより多く・長く見てもらい、**離脱を減らす**
3. スマートフォンでの**見やすさ**、**操作性**を最優先に考える (スマートフォン・ファースト)
4. **ユーザーの興味や関心に寄り添った**コンテンツを提供する

## 新たな指標

これまでは魅力的なサイト作りが重要視されてきたので、PV 数やセッション数が主な指標でした。これからはいかにユーザのニーズに合ったコンテンツを提供するか、そしてより長くサイトにとどまることが重要なので、次のような指標が大切です。

- ページごとの滞在時間
- 再訪問率
- 関連コンテンツへの遷移率
- SNS などでのシェア数

## その他所感

これまでの CMS に関する自分の理解は、CMS によっていろいろな人が関わることで、魅力的なサイトを素早く作ることができるな、というものでした。ただ、最近の動向に目を向けると、それだけではなく**ユーザーを第一に考えたときに** CMS をどのように活用するのか、延いてはどのような Web サイトであるべきかを常に考え続ける必要があるなと思いました。

また、次のことも大事だと思いました。

- マルチデバイス対応が必要。いかにこれを素早く実現するか
  - 接点が圧倒的に増えたことで、どのような方法であっても快適に閲覧できるようにしたい
- ユーザーの行動履歴とその分析
  - また訪問したい、と思ってもらえるような導線作りやコンテンツ提供をしていく
- API 連携
  - 様々なプラットフォームですぐ使えるように、即時更新のできる使い勝手のいいデータ連携が必要
  - 役割に合った必要最低限の API を作れるように
